评估 **Exploring Transformer Backbones for Heterogeneous Treatment Effect Estimation** 在 **IHDP100、IHDP1000、Jobs、TWINS** 四个 benchmark 上的效果，本质是做一个 **标准化的因果推断实验 pipeline**。关键是：**每个数据集的评价指标和实验 protocol 不一样**。下面给你一个可直接执行的完整流程。

------

# 一、总体评估流程

整个实验结构：

```text
for dataset in {IHDP100, IHDP1000, TWINS, Jobs}:
    for simulation in N_simulations:
        split data (train/val/test)

        train TransTEE
        early stop on validation

        evaluate on test set

    report mean ± std
```

最终输出表格：

| Dataset  | Metric      | TransTEE   |
| -------- | ----------- | ---------- |
| IHDP100  | √PEHE       | mean ± std |
| IHDP100  | ATE error   | mean ± std |
| IHDP1000 | √PEHE       | mean ± std |
| TWINS    | √PEHE       | mean ± std |
| Jobs     | Policy Risk | mean ± std |

### 2 实验步骤

IHDP100：

```
100 simulations
```

循环：

```

```

每个 simulation：

1️⃣ 读取数据

```
X
T
Yf
Ycf
```

计算：

```
Y1 = Yf if T=1 else Ycf
Y0 = Ycf if T=1 else Yf
```

真实ITE：

```
ITE_true = Y1 - Y0
```

------

2️⃣ 数据划分

推荐：

```
train 63%
val   27%
test  10%
```

------

3️⃣ 训练 TransTEE

输入：

```
X
T
```

输出：

```
ŷ1(x)
ŷ0(x)
```

预测ITE：

```
ITE_pred = ŷ1 - ŷ0
```

------

4️⃣ 计算指标

### PEHE

[
PEHE =
\frac{1}{N}\sum (ITE_{true}-ITE_{pred})^2
]

论文报告：

```
√PEHE
```

------

### ATE error

真实：

[
ATE = mean(Y1-Y0)
]

预测：

[
\hat{ATE} = mean(ŷ1-ŷ0)
]

误差：

```
ATE_error = |ATE - ATE_hat|
```

------

5️⃣ 存储结果

```
√PEHE_sim
ATE_error_sim
```

------

6️⃣ 汇总

最终：

```
mean ± std over 100 simulations
```

# 八、强烈推荐的复现结构

建议写统一实验框架：

```python
for dataset in datasets:

    for simulation in simulations:

        train_data, val_data, test_data = split()

        model = TransTEE()

        train(model)

        metrics = evaluate(model)

        results.append(metrics)

    report(mean(results), std(results))
```

------

下面是复现 **Exploring Transformer Backbones for Heterogeneous Treatment Effect Estimation** 时最关键、也是最容易导致结果严重偏差的 **5 个技术细节**。很多人复现 **IHDP / TWINS / Jobs** 时 PEHE 偏高（例如 2–3），基本都出在这里。

------

# 1 结果变量必须标准化（IHDP 最关键）

数据集：
IHDP Dataset

IHDP 的 outcome (Y) 分布通常 **均值很大、方差很大**。

如果直接训练：

```text
X,T → model → Y
```

Transformer 或 MLP 都会很难优化。

正确做法：

```text
Y_norm = (Y - mean(Y_train)) / std(Y_train)
```

只用 **train set 的统计量**。

训练时：

```text
loss = MSE(Y_pred_norm, Y_norm)
```

预测后需要 **反标准化**：

```text
Y_pred = Y_pred_norm * std + mean
```

否则计算：

```text
ITE = Y1 - Y0
```

会完全错。

**很多复现 PEHE=3+ 的原因就是没做这个。**

------

# 2 ITE 预测方式必须正确

模型输出一般是：

```text
ŷ1(x)
ŷ0(x)
```

ITE 计算：

```text
ITE_pred = ŷ1 - ŷ0
```

**但训练时只使用 factual loss：**

```text
if T == 1:
    loss = (ŷ1 - Y)^2
else:
    loss = (ŷ0 - Y)^2
```

很多人错误写成：

```text
loss = (ŷ1 - Y)^2 + (ŷ0 - Y)^2
```

这是错的，因为：

```text
counterfactual 不存在
```

------

# 3 PEHE 计算必须用 √PEHE

论文几乎全部报告：

```text
sqrt(PEHE)
```

而不是：

```text
PEHE
```

公式：

真实ITE：

```text
ITE_true = Y1 - Y0
```

预测ITE：

```text
ITE_pred = ŷ1 - ŷ0
```

PEHE：

[
PEHE = \frac{1}{N}\sum (ITE_{true}-ITE_{pred})^2
]

报告：

[
\sqrt{PEHE}
]

如果你直接报告 PEHE：

```text
数值会大很多
```

例如：

```text
√PEHE = 1.2
PEHE = 1.44
```

------

# 4 每个 simulation 必须单独训练

IHDP100：

```text
100 个 simulation
```

每个 simulation：

```text
covariates X 相同
treatment 不同
outcome 不同
```

正确做法：

```text
for sim in 1..100:

    train model

    evaluate
```

错误做法：

```text
把100个simulation拼起来训练
```

会导致：

```text
PEHE 完全错误
```

------

# 5 数据划分方式要一致

很多论文默认：

IHDP：

```text
train 63%
val   27%
test  10%
```

如果你用：

```text
train 80%
test 20%
```

会导致：

```text
early stopping 不稳定
```

Transformer 特别容易：

```text
overfit
```

------

# 6 TWINS 的隐藏坑（很多人忽略）

数据集：
TWINS Dataset

TWINS 的 treatment 是：

```text
lighter twin treated
```

但论文通常会 **随机生成 treatment**：

```text
P(T=1|X)
```

如果你直接用原始 treatment：

```text
ITE learning 会失效
```

必须使用论文提供的：

```text
synthetic treatment assignment
```

------

# 7 Jobs 的指标不同

数据集：
Jobs Dataset

Jobs 没有 counterfactual：

```text
不能算 PEHE
```

只能算：

```text
ATT error
policy risk
```

# 8 一个标准复现结构

建议写统一 pipeline：

```python
for dataset in datasets:

    for sim in simulations:

        X,T,Y = load_data()

        train,val,test = split()

        model = TransTEE()

        train_model(model)

        y1_hat, y0_hat = predict(model)

        ite_pred = y1_hat - y0_hat

        metrics = evaluate()

        store(metrics)

report(mean,std)
```
